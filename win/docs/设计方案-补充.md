# 你现在应该追加这个 Prompt

直接复制：

The breathing orb should dynamically adapt to the background behind it.

Problem:

- white orb disappears on bright backgrounds
- dark orb disappears on dark backgrounds

Solution:
Create an adaptive ambient overlay system.

Behavior:

- automatically detect the brightness and dominant color behind the orb
- dynamically adjust glow color, contrast, opacity, and blur
- maintain subtle visibility on any background

If background is dark:

- use soft white moonlight glow

If background is bright:

- use dark foggy glow
- slightly cool gray-blue tones

If background is colorful:

- softly adapt to the dominant color
- maintain low saturation
- preserve calm atmosphere

The orb should never feel like:

- a notification
- a cursor
- a button
- a floating app

It should feel like:

- breathing air
- adaptive moonlight
- environmental rhythm
- atmospheric presence

Visual direction:

- adaptive glow
- soft blur
- subtle fog
- context-aware contrast
- dynamic ambient lighting

Animation:

- extremely slow breathing
- soft opacity pulsing
- atmospheric motion
- barely noticeable

The overlay should intelligently blend into:

- dark IDEs
- white documents
- colorful websites
- videos
- desktop backgrounds

Avoid hard contrast.
Avoid pure black.
Avoid pure white.

Use intelligent adaptive tones instead.
